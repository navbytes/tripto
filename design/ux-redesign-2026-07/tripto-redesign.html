Tripto · screen redesign · v2

Four screens, rebuilt around the trip — not the form

Same brand (dusk indigo, warm amber, Fraunces + Sofia Sans), sharper structure. Home is now one list ordered by distance from now — shown twice, between trips and mid-trip, because a merged list finally has somewhere to put the trip you’re on. The signature is the rail: one timeline carrying time, timezone breaks and multi-night stays, with flights drawn as boarding passes. Every numbered pin maps to a note under the phone.

What changed

Home · one list, ordered by distance from now
  • 1The tabs are gone. One list: everything with an end date in the future, nearest first; then everything behind you, most recent first. Upcoming and past want opposite sort directions — that's the whole reason a single chronological sort doesn't work.
  • 2Register 1 — next. Full card, countdown ring, first departure. Only the nearest trip earns this.
  • 3The duplicate prompt moved onto the offender instead of sitting in a banner at the top. Identical dates means the pair is always adjacent, so the strip can fuse to the second card.
  • 4I dropped the “Today” divider I suggested. A live trip straddles it, so the marker starts lying the moment you're travelling. The archive header does the job better — it names the direction change instead of dating it.
  • 5Sticky year headers. Search sits in the archive header, where it's needed — not in the nav, where it would imply you search your next trip.
  • 6Register 3 — been. Rows, not cards. No countdown, no gradient, no avatars, muted thumbnail. Without this discipline a 2019 trip shouts as loudly as the live one.
  • 7Swipe a past trip → Copy to a new trip. A family that returns to the same places is exactly where a 6-year archive earns its keep. (Shown mid-swipe.)
  • 8Launch always opens at the top. Never restore scroll position on a list this long — the one thing you want is always the first thing.

Why the merge was worth it

Home · while you're actually travelling
  • 1Register 2 — now. Day 3 of 5 isn't upcoming and isn't past, so the old tabs had nowhere to put it. This is the app's best moment and it only exists once the list is one list.
  • 2Today's plan is inline on the card. Two items and a count — the answer to “what am I doing in an hour” without opening anything. Tapping lands on today in the itinerary, not day 1.

What changed

Itinerary · the timeline was fighting the timezone and the hotels
  • 1Title split: eyebrow Thailand, headline Bangkok & Khao Yai. No parenthetical, no two-line wrap, and Fraunces’ ampersand finally gets to be a feature.
  • 2Tabs stick under the status bar; day headers stick under the tabs. On a 7-day trip you always know where you are.
  • 3Traveller filter drops from ~90px to a 56px avatar row. The “Showing plans for” label was doing the job the chips already did.
  • 4Both hotels are booked for all 6 nights — that’s a conflict, not content. Say so once, at the top, and flag the offending cards.
  • 5The repeated “Staying at X — night 2 of 6” rows collapse into one muted strip under the day header. Lodging is context, not an event.
  • 6Timezone changes become rail markers — a hairline break in the timeline — instead of floating chips with no anchor.
  • 7Flights render as a boarding pass: both local times, duration, perforated footer for “lands 14:00 · clocks back 1h”. The arrival note belongs to the flight, not to the day. Day 1 now reads in true chronological order — the flight that brings you into Thailand comes before the car and hotel it precedes.
  • 8Empty days get an invitation, plus an optional suggestion card (new idea — kill it if you don’t want a recommendations surface).
  • 9The FAB was truncating hotel names. It becomes a contextual dock — “Add to Day 1” follows the day you’re looking at.

What changed

Add item · eleven stacked rows became one artifact
  • 1Paste first. The data is already in a booking email — typing it twice is the actual pain. Manual entry stays as the fallback, not the default path.
  • 2Type rail: equal tiles, one baseline (“Food” was sitting lower than its neighbours), verbs instead of nouns — Do / Eat / Ride reads as “what am I adding”.
  • 3Route as a boarding pass instead of eight stacked fields. From/To sit side by side with the duration computed between them — the same object as the itinerary card, so the form previews its own output.
  • 4Timezone lives directly under the time it modifies, not in a separate grey row whose association you had to infer.
  • 5“+1 day” is anchored to the arrival time as a badge. It was floating unattached in the right margin.
  • 6Seat, terminal, gate and code fold behind a disclosure with a summary line — optional most of the time, and it halves the scroll.
  • 7Traveller chips get a real selected state. Helper text explains the empty-means-everyone rule once, below the control.
  • 8Sticky save (disabled until valid) plus save & add the return leg — flights come in pairs.

What changed

Import · a report that ends in a dead end becomes one that ends in an action
  • 1Branded sheet, not raw system UI. This is the payoff moment of the whole onboarding — give it a headline and a heartbeat.
  • 2Stat tiles, not a settings list with right-aligned numbers. Fraunces numerals, tabular figures.
  • 343 travellers from 20 trips is a smell. The import knows it; say so and offer the merge before the person finds duplicate avatars scattered across their trips.
  • 4Every skipped row gets a recourse — Import anyway, Add dates. A list of things you refused to do, with no way to override, is a wall.
  • 5The notes limitation becomes a recovery path (a Gmail label) instead of an apology. Errors explain what happened and how to fix it.
  • 6One primary action out, and it names the reward: See your 20 trips.

Type

  • Fraunces 600 · trip names, headlines 34 / 32 / 30
  • Fraunces 600 · airport codes, stat numerals 24–32
  • Sofia Sans 700 · titles, times 14.5 / 13
  • Sofia Sans 600 · body, subtitles 12.5
  • Sofia Sans 800 · eyebrows, all-caps 10–11 / .14em

Every clock, date, seat and count uses tabular figures. A timeline where the digits don't line up is a timeline you don't trust.

Motion

  • Trip card → detail: the cover expands into the hero, the title travels. One shared element, nothing else moves.
  • Timezone marker draws its hairline left-to-right on first appearance in the viewport (140ms). Once. Never again.
  • Paste → fields populate top-to-bottom with a 30ms stagger, so you can see what it filled in.
  • Merge duplicates: the second card slides under the first and the stack settles. The undo toast lives for 6s.

Sort rule

One comparator, no special cases. A live trip falls out at position 0 on its own.

let ahead = trips
  .filter { $0.endsOn >= today }
  .sorted { $0.startsOn < $1.startsOn }   // nearest first

let been  = trips
  .filter { $0.endsOn < today }
  .sorted { $0.endsOn > $1.endsOn }       // most recent first

list = ahead + been                       // one section list

today is end-of-day in the trip's timezone, not the device's. Flying home on the 26th means the trip is still live at 23:00 in Naha even if your phone has already flipped over in Hong Kong.

Open questions

  • Is a split stay (two hotels, one trip, different nights) real for you? If so, the stay strip needs a two-line variant and the conflict rule needs a date-overlap test, not a same-trip test.
  • Is Bookings a filtered view of itinerary items, or its own record type? If it's a view, the boarding-pass card is shared and this all gets cheaper.
  • Where do traveller names come from — Gmail, manual entry, or an invite? The “add a name” affordance depends on the answer.
  • Covers: destination-seeded mesh (works offline, no licensing) or photos (needs a source and a fetch). I've assumed mesh.
